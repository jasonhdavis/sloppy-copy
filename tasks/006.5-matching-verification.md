# 006.5 — Author Matching Verification

Verifies that the author matching engine defined in [`tasks/006-author-matching-engine.md`](tasks/006-author-matching-engine.md) correctly assigns authors to evaluated stories.

Primary service: [`AuthorMatchingService`](authors/services.py:1)

---

## A. Preconditions

Stories must exist with:

```
status="evaluated"
```

Verify:

```
python manage.py shell
```

```
from stories.models import Story
Story.objects.filter(status="evaluated").count()
```

Expected:

```
> 0
```

Authors must exist:

```
from authors.models import Author
Author.objects.count()
```

Expected:

```
> 0
```

---

## B. CLI Verification

Run:

```
python manage.py match_authors --limit 3
```

Expected output:

```
Matching 3 stories
Story <id> matched with Author <id> score=<float>
```

CLI handler calls:

[`AuthorMatchingService.match_batch()`](authors/services.py:1)

---

## C. Database Validation

Inspect results:

```
Story.objects.filter(status="matched").values(
 "id",
 "matched_author",
 "matched_score"
)
```

Expected properties:

• `matched_author` populated

• `matched_score` between `0.0` and `1.0`

• `status="matched"`

---

## D. Tag Matching Validation

Confirm that overlap logic behaves correctly.

Example test:

Story tags:

```
["ai","policy","regulation"]
```

Author expertise:

```
["ai","machine-learning"]
```

Expected score:

```
1 / 3 = 0.33
```

Algorithm reference:

[`AuthorMatchingService.match_story()`](authors/services.py:1)

---

## E. Idempotency Test

Run command twice:

```
python manage.py match_authors --limit 5
python manage.py match_authors --limit 5
```

Second run expected:

```
0 stories processed
```

Because selector filters:

```
status="evaluated"
```

Selector reference:

[`get_evaluated_stories()`](stories/selectors.py:1)

---

## F. Failure Case Tests

### No Authors

Delete all authors and run:

```
python manage.py match_authors --limit 5
```

Expected:

• stories skipped

• status remains `evaluated`

---

### Empty Tags

Story evaluation returns empty tag list.

Expected:

```
match_score = 0
```

Story remains unmatched or lowest ranked author selected depending on implementation.

---

## G. Performance Check

Run for 20 stories:

```
python manage.py match_authors --limit 20
```

Expected:

• sequential processing

• deterministic ordering by `ai_score`

---

## H. Expected Database State

Before:

```
status="evaluated"
matched_author=NULL
```

After:

```
status="matched"
matched_author=FK
matched_score=FLOAT
```
