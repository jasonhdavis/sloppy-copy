# 005.5 — Story Evaluation Verification

Verifies that the evaluation engine defined in [`tasks/005-story-evaluation-engine.md`](tasks/005-story-evaluation-engine.md) correctly processes `Story` rows and stores deterministic AI outputs.

Primary service under test: [`StoryEvaluationService`](stories/services.py:1)
AI client entrypoint: [`AIClient.generate()`](ai/client.py:1)

---

## A. Preconditions

Before running verification:

1. Feed ingestion must have created `Story` rows.
2. Some stories must have:

```
status="ingested"
```

Verify via shell:

```
python manage.py shell
```

Example query:

```
from stories.models import Story
Story.objects.filter(status="ingested").count()
```

Expected:

```
> 0
```

---

## B. CLI Verification

Run evaluation command:

```
python manage.py evaluate_stories --limit 3
```

Expected console output:

```
Evaluating 3 stories
Story <id> evaluated score=<float>
```

Service invoked by CLI handler:

[`StoryEvaluationService.evaluate_batch()`](stories/services.py:1)

---

## C. Database Validation

Inspect evaluated stories:

```
python manage.py shell
```

Example check:

```
Story.objects.filter(status="evaluated").values(
    "id",
    "ai_score",
    "ai_rationale"
)
```

Expected properties:

• `ai_score` is float between `0.0` and `1.0`

• `ai_rationale` contains non‑empty text

• `status="evaluated"`

---

## D. JSON Output Validation

Ensure AI responses conform to schema returned by [`AIClient.generate()`](ai/client.py:1).

Expected structure:

```
{
  "score": float,
  "rationale": string,
  "tags": [string]
}
```

If parsing fails:

• service retries once

• story remains `status="ingested"` if failure persists

---

## E. Idempotency Test

Running the command twice should not re‑evaluate completed stories.

First run:

```
python manage.py evaluate_stories --limit 5
```

Second run:

```
python manage.py evaluate_stories --limit 5
```

Expected:

```
0 stories processed
```

Because selector filters:

```
status="ingested"
```

Selector reference:

[`get_unevaluated_stories()`](stories/selectors.py:1)

---

## F. Failure Case Tests

### Provider Failure

Simulate provider outage by invalid API key.

Expected behavior:

• exception caught

• story skipped

• status remains `ingested`

---

### Invalid JSON

Force malformed output in test provider.

Expected behavior:

1. retry once
2. log error
3. leave story unevaluated

---

## G. Performance Check

Run evaluation for 20 stories:

```
python manage.py evaluate_stories --limit 20
```

Expected behavior:

• sequential processing

• no memory spikes

• predictable runtime

---

## H. Expected Database State

Before evaluation:

```
status="ingested"
ai_score=NULL
```

After evaluation:

```
status="evaluated"
ai_score=FLOAT
ai_rationale=TEXT
```
